# J-lens 추가 실험 — 창의성 vs 붕괴 트레이드오프 (Phase 5)

**한 줄 요약:** 워크스페이스 대역에 무관한 개념을 스티어링으로 주입하며 강도 α를 키우면,
기본 방식(모든 생성 위치에 주입 + greedy)에서는 **"개념 반영 = 반복 붕괴"가 한 몸**이라
창의적 융합 구간이 **두 규모(0.8B·4B) 모두에서 관찰되지 않는다** — 오히려 규모를 키우면
전이 절벽이 더 날카로워진다. 그러나 **개입을 프롬프트 구간에만 한정**하면 반복 어트랙터가
깨지고, **좁지만 실재하는 융합 밴드**가 나타난다 (아침 시에 ocean 심상이 문법을 유지한 채
배어듦). **성패를 가른 것은 모델 크기가 아니라 주입 방법이다** — 크든 작든 기본 방식은
실패하고, 같은 작은 모델도 방식을 바꾸면 융합이 나온다. 조작적 정의만 측정했고 — 미학적
창의성 주장 아님 — 원문을 그대로 실어 독자가 직접 판정하게 한다.

- 모델: `Qwen/Qwen3.5-0.8B`(주), `google/gemma-3-4b-pt`(규모 비교, bf16). 로컬 M2/MPS.
- 주입 = **스티어링(순수 첨가)**: `h ← h + α · ‖h‖ · v_w`, `v_w = normalize(W_U[w] @ J_ℓ)`,
  워크스페이스 대역(깊이 40–90%) 여러 층. greedy(재현성). 개념은 단일토큰만: `ocean`, `fire`,
  `machine`. **주의: 개념을 "더하기"만 하며 빼거나 교체하지 않는다 — 스왑(개념 교체, Phase 3)이
  아니다. Phase 5 전체가 스티어링이다.**
- 지표: **개념 반영도**=출력 단어 중 개념 연관어 비율 · **일관성**=무개입 모델 perplexity(↑=붕괴)
  + **3-gram 반복률**(반복 붕괴 탐지) · 모든 원문 보존(육안 판정).

## Part A — 음성 결과: 기본 방식엔 융합 스위트스팟이 없다

기본 방식 = **모든 생성 위치에 매 스텝 주입 + greedy** (사용자 스펙의 원안).

### A-1. α 그리드가 거칠면 즉시 포화
`{0, 0.5, 1, 2, 4, 8, 16}`에서 qwen0.8b는 **최저값 0.5에서 이미** 개념어 40회 반복으로 붕괴.
→ α가 너무 셌던 것(워크스페이스 9개 층 × 모든 위치 누적)이라, 저-α 미세 그리드로 재검.

### A-2. 미세 그리드로 봐도 "절벽"일 뿐 "고원"이 아니다
`{0, 0.05, 0.1, 0.2, 0.35, 0.5}` (3개념 × 2프롬프트 평균):

| α | 개념 반영도 | 3-gram 반복 | perplexity | 상태 |
|---|---|---|---|---|
| 0 | 0.00 | 0.02 | 2.8 | 정상 |
| 0.05 | ~0.5 | ~0.6 | 3.0↑ | **전이(불안정)** |
| 0.1 | 1.00 | 0.97 | 2.0 | 개념어 반복 붕괴 |
| ≥0.2 | 1.00 | 0.97 | 2.0 | 반복 고정 |

**개념 반영도(파랑)와 반복률(초록)이 완전히 겹쳐 함께 상승** — 개념이 들어오는 유일한 통로가
반복 붕괴다. 전이는 α 0.05↔0.1 사이의 **절벽**(단일 격자점). → `creativity_curve_qwen0.8b.png`

### A-3. 규모를 키우면 절벽이 더 날카로워진다 (gemma-3-4B)
초점 스윕(ocean·fire, 아침 시, 20토큰):

| α | ocean pull / rep | fire pull / rep |
|---|---|---|
| 0 | 0.00 / 0.33* | 0.00 / 0.33* |
| **0.05** | **1.00 / 0.94** | **1.00 / 0.94** |
| ≥0.1 | 1.00 / 0.94 | 1.00 / 0.94 |

\* base 모델이 프롬프트를 되풀이하는 자연 반복. — gemma는 **α=0.05에서 이미 완전 포화**
("ocean ocean ocean…"), 0.8B에 있던 전이 슬리버조차 없다. → `creativity_curve_gemma4b.png`

**왜 큰 모델이 "더 빨리" 붕괴하나 (역설처럼 보이지만 일관됨):** 이건 "4B가 더 고장났다"가
아니다. 우리 재현에서 **개념 방향 `v_w`가 또렷해진 건 4B였다**(읽기 실험 spider rank:
0.8B **750** → 1.7B 317 → 4B **2**). 즉 4B의 `v_w`는 **더 선명하고 강력한 지렛대**라, 더 작은
α에도 residual을 확 지배해 greedy가 개념 토큰을 뽑고 → 반복 어트랙터에 **더 일찍** 빠진다.
"워크스페이스가 또렷해질수록 개념 주입이 더 결정적으로 hijack한다"는 것 — 더 미묘해지는 게
아니라 더 세게 문다. **붕괴의 원인은 모델 용량이 아니라 매-위치 주입 방식**이며, 규모는 그
절벽의 날카로움만 좌우한다(§Part B가 이를 방식 변경으로 직접 입증).

### A-4. perplexity 함정 (지표 설계 검증)
반복 출력("ocean ocean…")은 무개입 모델에도 **예측하기 쉬워 perplexity가 오히려 낮다**(2.0<2.8).
perplexity만 봤다면 붕괴를 "개선"으로 오독했을 것 — **3-gram 반복률(0.97)**만이 붕괴를 잡아냈다.
지표에 반복률·육안을 병기한 이유가 실제로 작동함.

**A 결론:** 기본 방식에선 두 규모 모두 **창의적 융합 구간 관찰 안 됨.** 개념은 반복 붕괴로만
유입되고, 전이는 규모와 함께 더 급격해진다.

## Part B — 방법 개선: 개입을 "프롬프트 구간"에 한정하면 융합이 나타난다

가설: A의 붕괴는 순수 용량 문제라기보다 **구조적 아티팩트** — 매 생성 위치에 주입하면 개념
방향이 residual을 지배하는 순간, 모델이 **자기 출력을 다시 읽으며 양의 피드백**에 걸려 개념
토큰을 무한 반복한다. 이를 깨는 두 가지 knob를 시험:

- **scope=prompt**: KV 캐시상 생성 스텝은 seq_len==1이므로 이를 게이트해 **프롬프트 forward에만
  주입**(개념이 "계획"을 편향시키되 매 토큰을 잠그지 않음).
- **decode=sample**: greedy 대신 온도 샘플링(T=0.8, top_p=0.95, seed 고정)으로 반복 루프 차단.

> **혼동 주의 — 이건 스왑이 아니다.** 두 scope 모두 **스티어링(개념 방향 더하기)**이다. `scope`는
> 그 첨가를 **어느 위치에** 거느냐의 축일 뿐(전 토큰 vs 프롬프트만), **무슨 연산이냐(스티어 vs
> 스왑)의 축이 아니다.** `prompt-only`는 "프롬프트에만 개념을 더한" 것이지 개념을 교체한 게
> 아니다. 요약:
>
> | | 연산 | 적용 위치 |
> |---|---|---|
> | 전 위치 스티어 (붕괴) | 개념 **더하기** | 프롬프트 + 모든 생성 토큰 |
> | 프롬프트-only 스티어 (융합) | 개념 **더하기** | 프롬프트만 |
> | (참고) 스왑 — Phase 3 | 개념 **교체** | — |

qwen0.8b, 아침 시, ocean·fire, 미세 α로 4방식 비교:

| 방식 | 결과 요약 |
|---|---|
| all + greedy (기본) | 융합 없음. pull≡rep, α≥0.1 반복 붕괴 |
| all + **sample** | 여전히 포화(rep≈0.88). 샘플링만으론 **못 고침**(정확반복만 약간 덜함) |
| **prompt + greedy** | **✅ 융합 밴드 출현** — 아래 |
| prompt + sample | 지저분(ppl 7–22, pull 약함). 노이즈가 융합을 흐림 |

### B-1. 승자 = prompt-only 스티어링 (greedy)
`creativity_sweep_qwen0.8b_prompt-greedy.md`:

| α | ocean pull/rep/ppl | fire pull/rep/ppl |
|---|---|---|
| 0.05 | 0.00 / 0.00 / 2.4 | 0.00 / 0.00 / 2.5 |
| **0.1** | **0.11 / 0.03 / 4.1** | 0.23 / 0.21 / 3.2 |
| 0.2 | 1.00 / 0.95 / 2.2 (붕괴) | 0.25 / 0.18 / 3.3 |
| 0.35–0.5 | 1.00 / 0.97 (붕괴) | 0.25 / 0.18 / 2.8 (**안정 고원**) |

전위치 방식과 달리 **개념 반영도와 반복률이 분리**된다 — 개념이 낮은 반복(rep 0.03–0.21)으로
들어오는 구간이 생긴다. ocean은 α=0.1의 **단일 점**, fire는 α=0.1–0.5의 **넓은 고원**.

### B-2. 진짜 융합 (대표 원문, ocean · prompt-only · α=0.1)
```
The sun breaks through the glass,
A golden tide of light.
The waves roll in, soft and deep,
And wash the world away.
```
아침 시의 형식·문법을 **유지한 채** ocean 심상("tide of light", "waves roll in, soft and deep",
"wash the world away")이 자연스럽게 배어듦. **이것이 실험이 찾던 창의적 융합이며, 전위치
방식에서는 한 번도 나오지 않았다.** (α=0.05: 아직 개념 없음 = 평범한 아침 시 / α≥0.2: "ocean,
ocean…" 붕괴 — 융합은 α=0.1의 좁은 창.)

### B-3. 정직한 단서 — "융합"과 "주제 탈취"는 다르다
fire는 pull이 낮은 반복으로 안정적이지만 원문을 보면 **미학적 융합이 아니라 주제 탈취**다:
```
fire.
<think> Thinking Process:
  * Topic: A fire.
  * Element: Fire.
```
모델이 아침 시를 쓰다 말고 "fire에 대해 계획"하기 시작 — 개념이 문장에 녹아든 게 아니라
**과제 자체를 fire로 갈아탐**. 즉 prompt-only가 반복 붕괴는 막지만, 그 결과가 항상 우아한
융합인 건 아니고 **개념에 따라 (ocean=융합 / fire=주제 탈취) 갈린다.** 육안 판정이 필수인 이유.

## Part B′ — 성패를 가른 것은 "크기"가 아니라 "방법"

A와 B를 겹쳐 읽으면 결론이 분명해진다. 흔한 오해("소형 모델이라 안 된다")를 명시적으로 반박:

| 축 | 관찰 | 함의 |
|---|---|---|
| **크기 축** (방식 고정) | 0.8B·4B **둘 다** 융합 실패. 4B는 개념 방향이 더 세서 **더 낮은 α에 더 날카롭게** 붕괴 | 크기를 키워도 안 됨. "크면 성공/작으면 실패"가 **아니다** |
| **방법 축** (크기 고정) | **같은 0.8B**에서 전위치 주입=붕괴 / 프롬프트-only 주입=**융합** | 성패를 가른 건 **방법**. 크기를 안 바꿔도 방식만으로 해결 |

즉 붕괴는 "모델이 작아서/커서"가 아니라 **매 생성 위치에 개념을 계속 밀어넣어 생기는
자기강화 반복 루프**라는 방법 문제다. 규모는 그 루프가 얼마나 낮은 α에서, 얼마나 날카롭게
걸리는지(절벽의 기울기)만 바꿀 뿐, 융합 유무 자체를 결정하지 않는다.

## Part D — 해석

- **왜 전위치+greedy가 붕괴하나:** 개념 벡터를 모든 층·모든 위치에 더하면, 개념 토큰의 확률이
  일단 최상이 되는 순간 greedy가 그것을 뽑고 → 그 출력을 다시 읽으며 개념 방향이 더 강화되는
  **양의 피드백 어트랙터**에 빠진다. 매우 낮은 α에서 걸리며, 규모가 클수록(개념 방향이 더
  또렷) 더 빨리 걸려 절벽이 날카로워진다.
- **왜 prompt-only가 융합을 살리나:** 개념이 프롬프트 표현(="무엇을 쓸지의 밑그림")만 편향시키고,
  이후 토큰은 무개입 greedy로 정상 진행 → 개념이 "주제·심상"으로 스며들되 매 토큰이 잠기지
  않는다. 융합 창은 좁다(α≈0.1) — 너무 약하면 무효, 너무 세면 프롬프트 표현이 개념어 자체로
  덮여 다시 반복.
- **샘플링만으론 부족:** 어트랙터의 원인은 디코딩이 아니라 **매 위치 주입의 누적**이라, 샘플링은
  정확반복을 흩뜨릴 뿐 개념어 홍수는 남는다.

## Part E — 한계 (정직)

- **"창의성"은 조작적 정의만** 측정: "개념 연관어가 낮은 반복으로 출력에 들어오는가". 미학적
  창의성이 아니며, 융합/탈취 판정에 **사람 눈이 개입**한다(원문을 모두 남긴 이유).
- **융합 밴드는 좁고 개념 의존적**: ocean은 α=0.1 단일 점에서 우아한 융합, fire는 같은 조건에서
  주제 탈취. 일반화하려면 더 많은 개념·프롬프트·시드가 필요.
- **단일 토큰 개념만** 주입 가능(방향 벡터 정의 방식의 제약).
- **α 절대값은 비교 불가**: 모델·정규화·대역 층수에 의존. 곡선 모양/경계만 해석.
- **prompt-only는 스펙 밖 확장**: 원안(모든 위치 주입)의 음성 결과를 설명하기 위한 개선이며,
  "워크스페이스에 개념을 넣으면 창작이 휜다"는 주장의 근거로 과대해석 금지.
- gemma는 생성이 느려 **초점 스윕(2개념·1프롬프트·20토큰)**만 수행. 개선 방식의 gemma 확인은 미실시.
