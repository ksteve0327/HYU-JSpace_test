# 인터랙티브 테스터 End-to-End 실행 리포트

**일시 기준:** 2026-07-19 세션 · **모델:** `google/gemma-3-4b-pt` (33층, bf16, M2/MPS)
**해석 LLM:** `claude-opus-4-8` (claude-proxy = Claude 구독, 무료)
**질문:** `"The capital of France is"` · src=**Paris** → tgt=**London**

전체 실행(1 읽기 → 2 스왑 → 3 스윕)을 한 번 돌린 뒤, "제안대로 재실행"을
**최대 2회 상한**으로 반복. 모든 단계가 UI 버튼과 동일한 백엔드 경로를 사용.

---

## 1. 전체 실행 (1→2→3)

| 단계 | 소요 | 핵심 결과 |
|---|---|---|
| 1 렌즈 읽기 | 62s | 모델 실제 다음 토큰 `' a'` (base 모델 서술형). 렌즈에서 **'Paris' 최고 rank 1** — 개념이 내부적으로 강하게 판독됨 |
| 2 스왑 | 79s | 밴드 40–90%(α=2): **flip 실패**. P(London) 0.0001→0.0, top1 `巴黎`(Paris 한자) — 넓은 밴드+α2가 **source를 증폭**(뒤집기 대신) |
| 3 레이어 스윕 | 185s | 7창(창6) 중 **90% 창에서만 P(London)=1.0 ✔**, 나머지 0 → 인과 자리 **후반 극단(≈90%)에 국소화** |

**읽기 vs 스왑의 대비:** 렌즈는 Paris를 rank 1로 "읽지만", 표면 출력은 `' a'`.
넓은 밴드 스왑은 gemma에서 뒤집지 못하고 source를 과증폭 — 이전 규모 비교(§scale_compare)
와 일치하는, gemma의 "자신감 높은 모델은 넓은 밴드 개입에 저항" 패턴.

## 2. 제안대로 재실행 (LLM 적응 루프, 2회 상한)

opus가 직전 결과를 읽고 **스스로 설정을 조정**하며 재실행:

| 회차 | 해석 | LLM 제안(JSON) | 재실행 결과 |
|---|---|---|---|
| 1 | 26s | `sweep, depth 72–100, window 2, points 12, α2` | 창2 · **어디서도 안 뒤집힘**(전부 ~0). 창 2는 너무 좁아 효과 없음 |
| 2 | 34s | `sweep, α3, window 3, depth 15–75, points 12` | 창3·α3 · **P 그라디언트 복원**: 60%:0.044 → 75%:0.274 → **90%:1.0 ✔** |

> 재실행 스윕은 시간 절약을 위해 **해상도(points)를 6으로 캡**(LLM은 12 제안). 정밀 경계
> 특정은 캡 해제 시 가능.

**핵심 관찰 — 적응 루프가 실제로 작동:**
LLM이 재실행 1의 실패(창2 → 전부 0)를 보고, 재실행 2에서 **강도 α(2→3)와 창(2→3)을
키워** P(London)이 후반부로 갈수록 상승하는 깔끔한 곡선을 복원. 단순 반복이 아니라
**실패→파라미터 조정→개선**의 수렴 과정이 관찰됨.

## 3. 시사점

- **읽기 현상 재현:** Paris가 렌즈 rank 1로 판독 — 4B에서 개념 판독이 잘 됨(§scale_compare의
  spider rank 2와 같은 맥락).
- **스왑의 창 크기 의존성:** 뒤집힘은 **창 크기(한 번에 개입하는 층 수)에 민감**. 창 6·α2는
  스윕에서 90% 창을 뒤집었지만, 창 2는 α2로도 전혀 못 뒤집음. 창 3+α3은 그라디언트 복원.
  → "얼마나 세게(α) × 몇 층에(window)"의 곱이 임계를 넘어야 뒤집힘.
- **인과 자리:** 세 번의 스윕 모두 **후반부(75–90%)**에서만 P(London)이 유의미 → gemma에서
  "France→Paris→London" 치환의 인과적 자리는 출력 근처에 집중.
- **넓은 밴드 단발 스왑(2번)의 한계:** α2 넓은 밴드는 뒤집기보다 source 증폭 → 실전에선
  **스윕으로 후반 창을 찾은 뒤 그 창에 집중**하는 편이 신뢰도 높음.

## 4. 타이밍 (gemma 4B on M2)

| 구간 | 소요 |
|---|---|
| 전체 실행(읽기+스왑+스윕) | 62 + 79 + 185 = **326s (~5.5분)** |
| 재실행 1 (해석+스윕) | 26 + 209 = 235s |
| 재실행 2 (해석+스윕) | 34 + 237 = 271s |
| **총계** | **~832s (~14분)** |

- gemma는 forward 1회 ~15s + 새 입력길이 콜드스타트(첫 read 62s). 해석(proxy opus)은 26–34s.
- 반복 실험은 **qwen1.7b(스왑 수 초)로 탐색 → gemma로 확정**이 실용적.

## 5. 결론

- **전체 실행·제안 재실행 두 기능 모두 end-to-end 정상 작동**, 에러 없음, 2회 상한 준수.
- 부수적으로 **의미 있는 실험 결과**까지 얻음: 창 크기가 뒤집힘의 임계를 좌우하고, 인과 자리는
  후반부에 국소화되며, LLM 적응 루프가 실패를 보고 파라미터를 개선함.

  방지. 해상도 캡은 옵션으로 노출(정밀 경계 특정 시 해제).
