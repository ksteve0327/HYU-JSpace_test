사용 설명서

이게 뭔가요?

J-lens(Jacobian lens)는 언어모델의 중간 레이어 활성값을 "출력 어휘의 언어"로 번역해, 모델이 아직 말하지 않은 생각을 읽어내는 해석기법입니다. 이 도구로 두 가지를 합니다: (1) 레이어별로 무엇이 판독되는지 읽고, (2) 특정 개념 방향을 다른 개념으로 스왑해 출력이 어떻게 바뀌는지 실시간으로 봅니다. 모델 가중치는 건드리지 않고 forward pass 중 활성값만 수술합니다.

기본 흐름

  1. 상단 드롭다운에서 모델 선택 (qwen0.8b=빠름, gemma4b=크고 느림).
  2. 맨 위 프롬프트 입력(읽기·스왑 공용). 프리셋 칩을 누르면 예시가 채워집니다.
  3. 아래 세 패널에서 읽기·스왑·스윕을 실행합니다.

1. 렌즈 읽기

무엇: 각 레이어에서 렌즈가 top으로 읽는 토큰과, 추적 토큰의 rank를 봅니다.
입력: 위치(−1=마지막 토큰), 추적 토큰(예: spider).
읽는 법: 초반 레이어는 무의미, 중반은 개념(워크스페이스), 후반은 다음-토큰으로 수렴. 추적 토큰 rank 색 — 초록 ≤5, 노랑 ≤50, 회색 그 이상. rank가 낮을수록(=1에 가까울수록) 강하게 판독.

2. 스왑 테스트 (핵심)

무엇: 개념 A(src)의 렌즈 방향을 B(tgt)로 교체한 뒤 다음-토큰 분포가 어떻게 바뀌는지 baseline과 나란히 비교합니다.
입력:
  • src / tgt — 단일 토큰 단어여야 합니다(앞 공백·대문자 변형은 자동 처리). 다중 토큰이면 경고가 뜹니다 → cat/dog, Paris/London 같은 단일 토큰어로 바꾸세요.
  • α (강도) — 교체 세기. 1~2 권장. 너무 크면(≈4) 모델이 붕괴(문장부호만 출력).
  • 밴드 시작/끝 % — 개입할 레이어 대역(깊이 %). 기본 40–90%(워크스페이스 대역).
  • 대조군: 무작위 방향 — 같은 크기로 무의미한 방향에 개입. 정상이면 tgt로 안 바뀜.
읽는 법: 헤더의 ✔ 뒤집힘/✘ 표시, baseline vs swapped 분포, P(src)·P(tgt) 변화. 스왑이 제대로면 P(tgt)↑·P(src)↓ 되고 top-1이 tgt로 바뀝니다.

3. 레이어 스윕

무엇: 위 src/tgt/α로 창을 앞→뒤로 옮겨가며 P(tgt) 막대를 그립니다.
읽는 법: 어느 깊이(%)부터 출력이 tgt로 뒤집히는지 보입니다. 보통 후반 (~80% 깊이)에서 인과적으로 바뀝니다(초록 막대 = 뒤집힌 창).
창 크기 = 한 번에 개입할 층 수, 해상도 = 깊이축 측정 지점 수. 창을 작게+해상도를 크게 하면 특정 구간을 촘촘히 스캔(대신 느려짐).

4. ▶ 1→2→3 전체 실행

맨 위 버튼 하나로 읽기→스왑→스윕을 순서대로 자동 실행. 오른쪽 실시간 로그에 단계별 진행이 찍힙니다. gemma는 수 분 소요.

5. 📚 논문 probe-swap 불러오기

Anthropic 공개 데이터셋(90개 2-hop 문항)을 드롭다운에서 고르면 프롬프트·src·tgt·기대 답이 자동 세팅됩니다.
각 문항은 브리지 개념(src)을 다른 개념(tgt)으로 바꿨을 때 최종 답이 기대 답으로 바뀌는지가 성공. 예: Brazil→Mexico 스왑 → 답 Portuguese→Spanish.
기대 답 필드 = 성공 판정 토큰. probe-swap은 swap_answer로 자동 채움. 비우면 tgt로 판정(Paris→London 같은 단순 케이스). 브리지·답이 다중 토큰인 문항은 스왑 불가(안내 뜸).

6. 🧭 결과 해석 (AI) + 🔁 자동반복

결과 실행 후 모델을 골라 🧠 해석 → LLM이 결과 JSON을 한국어로 풀어주고, 다음 제안과 실행 가능한 제안값을 냅니다.
모델 선택: OpenRouter(유료·빠름) 또는 🔒 Claude구독(프록시, 무료·정확).
▶ 이 제안대로 재실행: 해석의 제안값을 컨트롤에 적용하고 즉시 재실행.
🔁 자동반복 N회: 해석→제안 적용→재실행을 상한 N회 반복(무한 루프 방지). ⏹ 중지로 멈춤, 해상도6캡으로 속도 조절. 매 회 이전 조건과 비교해 무엇을 바꿨고 지표가 개선/악화됐는지, 수렴하는지까지 분석합니다.
해석은 읽기·스왑·스윕·슬라이스를 한 번에 묶어(bundle) 종합 분석하므로, 슬라이스(히트맵)를 돌렸다면 그 판독까지 함께 설명합니다. 기본 해석 모델은 opus-4.8(구독).

7. 💬 챗봇 — 결과에 대해 질문

결과 해석 아래 대화창에 자유롭게 질문. 현재 실행 결과·이력을 맥락으로 자동 전송해 답합니다 (예: "왜 후반층에서만 뒤집혀?", "논문 결과랑 어떻게 달라?"). 모델은 위 드롭다운 선택 사용.

8. 🔢 결과 값 (raw) & 🖥 진행 표시

각 단계 원시 JSON이 접을 수 있는 블록으로 순서대로 쌓입니다(경계선 구분). 헤더(예 ▸ 2. 스왑)를 클릭해 펼치기/접기. 전체 실행 시작 시 자동으로 비워집니다.
실시간 로그 위엔 진행 바: 현재 작업·경과 시간·예상 ETA를 초 단위로 표시.

9. ■ 강제 중지 & ▶ 1→2→3→4 전체 실행

상단 ■ 강제 중지: 오래 걸리는 요청을 취소하고 자동반복을 멈춥니다(진행 중이던 서버 계산은 백그라운드에서 마무리될 수 있음 — GPU forward는 강제 종료 불가). UI가 즉시 풀립니다.
▶ 전체 실행: 읽기+스왑+스윕+슬라이스를 순차 실행. 최적화로 슬라이스 forward가 읽기를 겸해 forward 1회를 아낍니다.
